# 프로젝트 개요: BizTone 변환기

BizTone 변환기는 특히 신입 전문가들이 일상적인 언어를 다양한 대상(상사, 동료, 고객)에게 적합한 비즈니스 커뮤니케이션 톤으로 변환할 수 있도록 돕기 위해 설계된 AI 기반 웹 애플리케이션입니다. 이 솔루션은 조직 내 의사소통 효율성과 전문성을 향상시키는 것을 목표로 합니다.

## 주요 기능

*   **대상별 톤 변환:** 텍스트를 공식적인(상사), 협업적인(동료), 또는 매우 정중한(고객) 톤으로 변환합니다.
*   **실시간 변환:** 빠른 변환 결과(평균 3초 이내 목표)를 제공합니다.
*   **텍스트 비교:** 원본 텍스트와 변환된 텍스트를 나란히 표시하여 쉬운 비교 및 학습을 돕습니다.
*   **복사 기능:** 변환된 텍스트를 클립보드에 쉽게 복사할 수 있도록 합니다.
*   **반응형 디자인:** 데스크톱, 태블릿, 모바일 폰을 포함한 다양한 기기에 최적화되어 있습니다.

## 아키텍처 및 기술

이 프로젝트는 명확한 프론트엔드 및 백엔드 구성 요소를 갖춘 고전적인 클라이언트-서버 아키텍처를 따릅니다.

### 프론트엔드

*   **기술:** HTML5, CSS3 (스타일링을 위한 Tailwind CSS), JavaScript (ES6+).
*   **역할:** 텍스트 입력, 대상 선택을 위한 사용자 인터페이스를 제공하고 변환 결과를 표시합니다. Fetch API 호출을 통해 백엔드와 통신합니다.

### 백엔드

*   **기술:** Python 3.11, Flask (웹 프레임워크), Flask-CORS (교차 출처 요청용), python-dotenv (환경 변수 관리용), Groq AI API 클라이언트.
*   **역할:** 프론트엔드에서 텍스트와 대상 독자를 받아 Groq AI API를 사용하여 처리하고 변환된 텍스트를 반환하는 RESTful API(`/api/convert`)를 호스팅합니다. 백엔드는 또한 정적 프론트엔드 파일을 제공합니다.
*   **AI 모델:** 현재 텍스트 변환을 위해 `llama-3.3-70b-versatile` 및 `llama-3.1-8b-instant`를 사용하도록 구성되어 있습니다(초기 PRD에서는 `moonshotai/kimi-k2-instruct-0905`를 지정했음).

### 배포 (계획)

*   **플랫폼:** 정적 파일 호스팅 및 서버리스 함수를 위한 Vercel.
*   **버전 관리:** `main`, `develop`, `feature` 브랜치 전략을 따르는 Git 및 GitHub.

## 시작하기

### 전제 조건

*   Python 3.11+
*   `pip` (Python 패키지 설치 관리자)
*   Groq API 키

### 설정 및 설치

1.  **리포지토리 클론:**
    ```bash
    git clone <repository-url>
    cd biztalk_llm # 또는 프로젝트 루트 디렉토리
    ```

2.  **백엔드 환경 설정:**
    ```bash
    python3 -m venv .venv
    source .venv/bin/activate # Windows에서는 `.venv\Scripts\activate` 사용
    pip install -r backend/requirements.txt
    ```

3.  **Groq API 키 구성:**
    프로젝트 루트 디렉토리(backend/ 및 frontend/ 옆)에 `.env` 파일을 생성하고 Groq API 키를 추가합니다.
    ```
    GROQ_API_KEY="YOUR_GROQ_API_KEY_HERE"
    ```

### 애플리케이션 실행

1.  **백엔드 서버 시작:**
    가상 환경이 활성화되어 있는지 확인하십시오(`source .venv/bin/activate`).
    ```bash
    python3 backend/app.py
    ```
    Flask 백엔드 서버가 시작되고 `http://127.0.0.1:5000`에서 접근할 수 있습니다.

2.  **프론트엔드 접근:**
    웹 브라우저를 열고 `http://127.0.0.1:5000`으로 이동합니다. 프론트엔드 인터페이스는 Flask 애플리케이션에 의해 제공됩니다.

## 개발 규칙

*   **프로젝트 구조:** 프로젝트는 Flask 애플리케이션을 위한 `backend/`와 정적 웹 자산을 위한 `frontend/`로 구성됩니다.
*   **API 키 관리:** 민감한 API 키(예: Groq API 키)는 클라이언트 측 노출을 방지하기 위해 환경 변수를 통해 관리되며, 배포 시 안전하게 처리될 예정입니다.
*   **스타일링:** Tailwind CSS는 반응형 및 유틸리티 우선 스타일링에 사용됩니다. 현재 설정은 개발용 CDN 버전을 사용하며, 프로덕션용으로는 PostCSS 플러그인 또는 Tailwind CLI를 통해 통합하는 것이 권장됩니다.
*   **버전 관리:** 기능 브랜치, 풀 리퀘스트 및 설명적인 커밋 메시지에 중점을 둔 표준 Git 워크플로우를 사용합니다.
